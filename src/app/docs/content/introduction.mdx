DX is a unified ecosystem of developer tools built around a shared philosophy: your tools should work together, not against each other.

## What is DX?

DX brings together code editing, build systems, web frameworks, AI agents, media pipelines, search engines, and CLI tools into a cohesive, interconnected platform. Instead of stitching together a dozen unrelated tools with different conventions, DX provides a consistent experience across your entire development workflow.

The ecosystem includes 20+ Rust-based tools spanning:

- **DX Code** — GPU-accelerated code editor (forked from Zed)
- **DX CLI** — Central command-line interface to all tools
- **DX Web** — Rust web framework with TSX authoring
- **DX Agent** — AI agent runtime with 25+ messaging channels
- **DX Flow** — Local-first AI workflow automation
- **DX Forge** — Content-addressed version control & build system
- **DX Style** — Rust-native CSS engine
- **DX Serializer** — Token-optimized data serialization
- And many more

## Key principles

**Unified toolchain.** Every tool shares conventions, formats, and integration points. Your CLI commands work with your editor, which talks to your build system, which deploys through your web framework.

**Local-first.** DX runs on your machine. No cloud dependency for core operations. Data stays where you control it.

**Extensible by design.** From MCP to DCP to WASM plugins, every tool exposes integration surfaces. Build on top of DX instead of fighting it.

**Performance-obsessed.** Written in Rust. Zero-copy serialization. SIMD-accelerated search. GPU-accelerated editing. Milliseconds matter.
